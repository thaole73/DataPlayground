Predicting stock returns is a complex yet important task. Past performance of individual stock can be a good predictor of future performance, fundamental factors such as earnings, cash flow, and management decisions should be considered when forecasting returns. Furthermore, quantitative methods such as portfolio optimization and risk analysis provide useful assessments to aid in the decision-making process (Avramov & Chordia, 2006). In this report, we use Compustat and CRSP datasets to run time-series regression models to see if existing variables, such as individual stock returns, size factors, or other macro factors, have predictive power over future stock returns. The goal is to leverage these findings to develop machine learning models to predict future stock returns.

I. Business Objective

In this report, we will aim to build four machine learning (ML) models including SGD based Linear Regression, Random Forest, KNN, and XGBoost for forecasting stock return based on fundamental analysis and technical analysis. We use financial indicators such as assets, liability, market cap, book value, earning per share, payable, etc. and technical factors such as historical return/price/volume as predictors. Historical price to earnings per share and book to market are also created as predictive variables. After building four ML models, we will calculate performance metrics to evaluate their predictive power.

2.2 Missing values and Outliers

The dataset retrieved from Compustat and CRSP contains many missing values. These missing values can bias the machine-learning models and reduce the prediction accuracy. In many cases, a dataset with missing values leads to invalid results. As our dataset is large with many features, we decided to drop feature columns that are populated mainly by missing values. Besides, for feature columns with sparse null values, deleting these values is a better option to avoid bias.

Outliers are data points that differ significantly from other data points in dataset. An outlier can cause serious problems in analysis and lead to incorrect results. Therefore, it is important to detect and treat these outliers to make our Machine Learning model perform better. In this report, we will use Z-score to detect and remove outliers. The concept of Z-score is to indicate how far away a certain point is from the mean. If Z-score of data point is greater than 3, we can conclude that our data point is an outlier (Ansoleaga, 2021)

{% highlight ruby %} # Function to detect outliers def zscore(df): for col in df.columns: df['Zscore'] = stats.zscore(df[col]) df= df[(df['Zscore'] < 3) & (df['Zscore'] > -3)] df.drop('Zscore', axis=1, inplace=True) return df {% endhighlight %}

2.3 Trend stationarizing feature

For time-series dataset, it is important to determine if a time-series is stationary or not. The concept of stationarity means the time series has no trend or no seasonal pattern. The distribution of a time series is the same with respect to time. The forecast of models on stationary time series tend to produce better prediction. (Chaudhari, 2021). As many features from our dataset have noticeable trends throughout the time, we stationarizes the features by calculating the percentage change between consecutive observations.

{% highlight ruby %} # Add new independent variables df['vol6'] = df['volume1'].rolling(6).mean().shift(1) df['vol12'] = df['volume1'].rolling(12).mean().shift(1) df['price6'] = df['price1'].rolling(24).mean().shift(1) df['price12'] = df['price1'].rolling(48).mean().shift(1) # Add new stationary variables df['delta_ret1'] = (df['ret1'] - df['ret1'].shift(1))*100/df['ret1'].shift(1) df['delta_ret2'] = (df['ret2'] - df['ret2'].shift(1))*100/df['ret2'].shift(1) df['delta_marketcap1'] = (df['marketcap1'] - df['marketcap1'].shift(1))*100/df['marketcap1'].shift(1) df['delta_price1'] = (df['price1'] - df['price1'].shift(1))*100/df['price1'].shift(1) df['delta_asset1'] = (df['asset1'] - df['asset1'].shift(1))*100/df['asset1'].shift(1) df['delta_liability1'] = (df['liability1'] - df['liability1'].shift(1))*100/df['liability1'].shift(1) df['delta_book_value1'] = (df['book_value1'] - df['book_value1'].shift(1))*100/df['book_value1'].shift(1) df['delta_EPS1'] = (df['EPS1'] - df['EPS1'].shift(1))*100/df['EPS1'].shift(1) df['delta_book_to_market1'] = (df['book_to_market1'] - df['book_to_market1'].shift(1))*100/df['book_to_market1'].shift(1) df['delta_PE1'] = (df['PE1'] - df['PE1'].shift(1))*100/df['PE1'].shift(1) df['delta_volume1'] = (df['volume1'] - df['volume1'].shift(1))*100/df['volume1'].shift(1) df['delta_payable1'] = (df['payable1'] - df['payable1'].shift(1))*100/df['payable1'].shift(1) df['delta_value_weight1'] = (df['value_weight1'] - df['value_weight1'].shift(1))*100/df['value_weight1'].shift(1) df['delta_vol6'] = (df['vol6'] - df['vol6'].shift(1))*100/df['vol6'].shift(1) df['delta_vol12'] = (df['vol12'] - df['vol12'].shift(1))*100/df['vol12'].shift(1) df['delta_price6'] = (df['price6'] - df['price6'].shift(1))*100/df['price6'].shift(1) df['delta_price12'] = (df['price12'] - df['price12'].shift(1))*100/df['price12'].shift(1) {% endhighlight %}

3.2 SGD based Linear Regression

Linear Regression is a machine learning algorithm based on supervised learning. SGD stands for Stochastic Gradient Descent: the gradient of the loss is estimated each sample at a time and the model is updated with a decreasing strength schedule (aka learning rate). This model comes in handy when we need to fit a large sample of data with many features. There are two main classes of SGD based model, SGD Classifier and SGD Regressor. This project will use SGD based Linear Regressor to predict stock return. To get better performance, we will find out the best set of parameters for SGD based Linear Regression model, and then use these optimal parameters to build model and make prediction.

{% highlight ruby %} # SGD Based Model param_grid = { "alpha": [1e-5, 3e-10, 1e-10], "eta0": [1e-5, 3e-10, 5e-13], } SGD_LR = SGDRegressor(loss='squared_loss', learning_rate='constant', penalty='l2', max_iter=1000) grid_search = GridSearchCV(SGD_LR, param_grid, cv=10, scoring='r2') grid_search.fit(X_train, y_train) print(grid_search.best_params_) # Best parameters SGD_LR_best = grid_search.best_estimator_ y_pred_LR = SGD_LR_best.predict(X_test) print("R2: {}".format(r2_score(y_test, y_pred_LR))) print("MAE: {}".format(mean_absolute_error(y_test, y_pred_LR))) print("MSE: {}".format(mean_squared_error(y_test, y_pred_LR))) {% endhighlight %}

3.3 Random Forest

Random Forest algorithm is a supervised machine learning model that is made of various decision trees. It grows multiple decision trees on various sub-samples of the dataset and then pick the average to improve the prediction accuracy and reduce risk of overfitting. This report will build Random Forest Regression Model to predict stock return. We will tune the random forest to choose best parameters for this model

{% highlight ruby %} # Tuning random forest model param_grid ={'max_depth': [40, 60, 80, 90, 100], 'max_features': [5, 7, 10], 'min_samples_leaf': [3, 5, 7], 'min_samples_split': [7, 10]} RF = RandomForestRegressor(n_estimators = 250, random_state = 42, n_jobs= -1) grid_RF = GridSearchCV(RF, param_grid, cv=5, n_jobs=-1, scoring = 'r2') grid_RF.fit(X_train, y_train) # Best parameters print(grid_RF.best_params_) RF_best = grid_RF.best_estimator_ y_pred_RF = RF_best.predict(X_test) print("R2: {}".format(r2_score(y_test, y_pred_RF))) print("MAE: {}".format(mean_absolute_error(y_test, y_pred_RF))) print("MSE: {}".format(mean_squared_error(y_test, y_pred_RF))) # Random Forest model after tuning RF_best = Pipeline([('rf', RandomForestRegressor(n_estimators=250, max_features=5, max_depth= 40, min_samples_split= 7, min_samples_leaf=7))]) RF_best.fit(X_train,y_train) y_pred_RF = RF_best.predict(X_test) print("R2: {}".format(r2_score(y_test, y_pred_RF))) print("MAE: {}".format(mean_absolute_error(y_test, y_pred_RF))) print("MSE: {}".format(mean_squared_error(y_test, y_pred_RF))) {% endhighlight %}

3.5 KNN

KNN (K-Nearest Neighbors) is an algorithm for supervised learning. It is a non-parametric, lazy learning algorithm used for classification and regression. It is a type of instance-based learning or non-generalizing learning: it does not attempt to construct a general internal model, but simply stores instances of the training data. When a prediction is required, the k-nearest training examples are located, and their class labels are aggregated to form a prediction
